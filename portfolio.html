// 01 — Overview

프로젝트 소개

Nextage는 의류 수선·리폼을 원하는 고객(Customer)과 이를 전문으로 하는 업체(Business)를 연결하는 역경매 방식의 B2C 중개 플랫폼입니다.


기존 수선·리폼 시장의 불투명한 가격 구조와 정보 비대칭 문제를 해결하기 위해, 고객이 의뢰글을 등록하면 전문 업체들이 경쟁 입찰 방식으로 견적을 제출하고, 고객은 가격·포트폴리오·리뷰를 비교해 최적의 업체를 선택할 수 있는 구조입니다.


카테고리는 2D Clothes / 3D Clothes / 3D Object / Metal 4가지로 구성되며, KIT SHOP을 통한 수선 키트 쇼핑 기능도 함께 제공합니다.

📋
역경매 의뢰 시스템
고객이 의뢰글 등록 → 업체들이 경쟁 입찰 → 고객이 최적 업체 선정
💳
포트원 v2 결제 연동
입찰 선정 결제 + KIT SHOP 결제, 금액 검증 및 자동 환불 처리
💬
실시간 WebSocket 채팅
STOMP 기반 1:1 채팅, 미읽음 수 실시간 배지, 이미지 첨부 지원
🔐
이중 보안 구조
Spring Security 이중 FilterChain + Interceptor 역할 기반 URL 접근 제어

// 05 — Core Logic

핵심 로직 흐름

01
업체가 의뢰글에 입찰 제안 등록
희망가격의 95% 이상 & 기존 최저가 이하 조건 검증. 동일 업체 중복 입찰 방지. 4단계 스텝 위자드 UI.
02
고객이 최대 3개 업체 비교 모달 조회
가격·예상완료일·A/S·설명 나란히 비교. 최저가 업체 자동 강조(👑 최저가 배지). 업체 포트폴리오 이동 링크.
03
고객이 원하는 업체 선정 클릭
치수 입력(Step 1) → 최종 확인(Step 2: 의뢰정보·업체제안·배송지 표시) → 결제 동의.
04
포트원 결제 완료 후 서버 금액 검증
DB 저장 금액과 결제 금액 비교. 불일치 시 FAILED 처리 및 실패 응답 반환.
05
트랜잭션 — 낙찰 일괄 처리
SELECT FOR UPDATE 동시성 제어 → bid SELECTED + 나머지 bid REJECTED + request PAID + dimensions(치수) 저장 — 원자적 처리.
06
업체 수락 시 채팅방 자동 생성
accept_status = ACCEPTED → chat_room 자동 생성 → request.status = SELECTED → 고객·업체 1:1 채팅 시작.
01
사전 주문 생성 (Order Pre-create)
ORDER_YYYYMMDDHHMMSS_XXXX 형식의 주문번호 발급. orders 테이블에 READY 상태로 저장.
02
포트원 v2 결제창 호출
Browser SDK로 카카오페이 등 결제창 호출. 결제 취소 시 처리 없이 반환.
03
서버 측 금액 검증 (Verify)
impUid + orderNo + totalAmount 서버 전달 → DB 저장 금액과 대조. 불일치 시 FAILED 처리.
04
트랜잭션 — 결제 완료 처리
payment_status = PAID 업데이트 → order_items 저장 → kit 재고 차감 → 장바구니 삭제 — 원자적 처리.
05
취소/거절 시 자동 환불
포트원 v2 Cancel API (Bearer Auth) 호출 → 성공 시 DB payment_status = CANCELLED 업데이트.
01
HTTP 요청 → Spring Security FilterChain
Business FilterChain(Order=1): /business/**, /auth/business/** 담당. Customer FilterChain(Order=2): 나머지 전체 담당. 비로그인 접근은 Security가 차단.
02
RoleRedirectInterceptor.preHandle() 실행
SecurityContextHolder에서 Authentication 객체 추출. 비로그인(anonymousUser)이면 통과 처리.
03
권한명으로 업체/고객 판별
getAuthorities()에서 ROLE_B로 시작하는 권한이 있으면 업체(isBusiness=true), 없으면 고객(isBusiness=false).
04
반대 영역 URL 접근 시 차단
업체가 /customer/** 접근 → /business/main?accessDenied=true 리다이렉트. 고객이 /business/** 접근 → /customer/main?accessDenied=true 리다이렉트.
05
공용 허용 경로 예외 처리
/customer/request/**: 업체도 의뢰 열람 허용. /business/portfolio/**: 고객도 포트폴리오 열람 허용. 정적 리소스(/css, /js, /images)는 WebConfig에서 제외.
01
업체가 배송완료 처리
deliveryStatus = 4 (배송완료) 로 변경 요청. POST /business/order/delivery/{orderId} 호출.
02
주문 완료 시각 기록
orders.due_date = 현재 시각으로 업데이트. 실제 완료일 자동 기록.
03
의뢰 상태 COMPLETE 처리
bid → request_id 조회 후 request.status = COMPLETE 업데이트.
04
정산 자동 생성 (중복 방지)
settlement 테이블에 해당 orderId 존재 여부 확인 후 없을 때만 생성. 매출액 / 수수료(10%) / 정산액(90%) 자동 계산 후 저장.
